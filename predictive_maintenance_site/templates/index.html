02 — Dataset

What Data We Have

The AI4I Predictive Maintenance Dataset contains 10,000 operational snapshots from a simulated milling machine. Each row captures the machine's state at one point in time, labelled as failure or no-failure.

🌡
Air Temperature [K]
Ambient air temperature around the machine. Generated using a random walk process around 300 K. Affects cooling efficiency and thermal environment of the machine.
295 – 304 K
🔥
Process Temperature [K]
Internal machine operating temperature. Always 8–10 K higher than air temperature due to heat generation. Thermal stress on components rises sharply above threshold.
305 – 314 K
⚙️
Rotational Speed [rpm]
Spindle speed in revolutions per minute. Very high or very low speeds indicate operational stress or tool wear failure modes.
1168 – 2886 rpm
🔩
Torque [Nm]
Rotational force applied to the cutting tool. High torque combined with low rotational speed is a strong indicator of mechanical overload and overstrain failure.
3.8 – 76.6 Nm
🪛
Tool Wear [min]
Cumulative minutes the cutting tool has been in use. The primary aging indicator — tools beyond 200 min fail more frequently across all failure types.
0 – 253 min
🎯
Target (Label)
Binary classification label. 0 = No Failure, 1 = Failure. Derived from five sub-failure types: tool wear, heat dissipation, power, overstrain, and random failures.
0 or 1

What the boxplots reveal

Rotational speed has a left skew — most machines operate in the 1200–1800 rpm range but a few run much faster. Torque shows high-end outliers beyond 65 Nm, which often correspond to overstrain failures. Tool wear is near-uniform since it increments linearly with usage time. These outliers are intentionally kept — they carry critical failure signal and removing them would reduce model sensitivity.

Key relationships discovered

Air Temp ↔ Process Temp (r = 0.88): Strong positive correlation. When ambient temperature rises, internal machine temperature follows — increasing thermal stress risk.

Rotational Speed ↔ Torque (r = −0.87): Strong negative correlation. This is mechanical law — power = speed × torque. Extreme combinations (very low speed + very high torque) trigger overstrain failures.

Tool Wear ↔ Target (r = 0.09): Weak but real. Tool wear amplifies other failure modes rather than causing failures on its own.

04 — ⚠ Problem Detected

Class Imbalance — The Critical Problem

During EDA, we discovered that the dataset is severely imbalanced. This is one of the most dangerous problems in real-world ML classification tasks.

Class Distribution — Original Dataset

The accuracy paradox

A model that always predicts "No Failure" achieves 96.6% accuracy without learning anything. This is why we must use F1-score and recall as primary metrics — not accuracy.

💥 Real-world consequence

In predictive maintenance, a missed failure (false negative) costs far more than a false alarm. An undetected failure can cause catastrophic equipment damage, production halt, or even worker injury. A biased model optimising for accuracy is useless — it would miss 100% of failures. We must fix the imbalance before training.

05 — Solution

SMOTE — Fixing the Imbalance

Synthetic Minority Oversampling Technique (SMOTE) generates new synthetic samples for the minority class by interpolating between existing examples — preserving distribution while balancing classes. We used random_state=42 for reproducibility.

Feature distributions remain structurally similar before and after SMOTE — confirming that synthetic samples follow the same statistical patterns as real data. 07 — Normalization

StandardScaler — Preparing for Training

Features exist at vastly different numerical scales. StandardScaler transforms each feature to zero mean and unit standard deviation so no feature dominates others due to magnitude alone.

Feature Ranges — Before Normalization
Feature Ranges — After Normalization
Normalized Feature Distributions
Why normalization matters here

KNN computes similarity using Euclidean distance. Without normalization, rotational speed (range ~1700) dominates over torque (range ~73) purely due to scale. After StandardScaler, all five features contribute equally to distance calculations.

Logistic Regression uses gradient descent to fit coefficients — unnormalized features cause slow convergence and biased weights.

Random Forest and Decision Tree are scale-invariant by nature but normalization ensures consistent preprocessing across the entire pipeline.

09 — Innovation

Hybrid Model — Best of Both Worlds

We combined the top two models using a Hard Voting Classifier (sklearn VotingClassifier). Each model independently predicts the class and the majority vote determines the final output — reducing the individual weaknesses of each model.

Why does combining models help?

RF is strong at precision — it avoids false alarms by being conservative. KNN is strong at recall — it catches almost all true failures by considering local feature space proximity. When RF misclassifies a borderline failure case, KNN's proximity-based vote often corrects it, and vice versa. The combination produces a classifier that is simultaneously more precise and more sensitive than either model alone — achieving the best F1 score of 97.5%.
