02 — Dataset

What Data We Have

The AI4I Predictive Maintenance dataset contains 10,000 samples with real machine sensor readings. Each row represents one operational snapshot of a machine.

🌡
Air Temperature
Ambient air temp around the machine. Indicates cooling efficiency and thermal environment.
Range: 295–304 K
🔥
Process Temperature
Internal operating temperature. Higher values signal thermal stress on components.
Range: 305–314 K
⚙️
Rotational Speed
RPM of the machine. Directly reflects operational load and mechanical wear.
Range: 1168–2886 rpm
🔩
Torque
Rotational force. High torque at low speed indicates excessive mechanical stress.
Range: 3.8–76.6 Nm
🪛
Tool Wear
Cumulative usage time of the cutting tool. A primary aging indicator for failure prediction.
Range: 0–253 min
🎯
Target (Label)
Binary classification: 0 = No Failure, 1 = Failure. This is what we predict.
0 or 1

04 — ⚠ Problem Detected

Class Imbalance — The Big Problem

During EDA, we discovered a critical issue: the dataset is severely imbalanced.

💥 Why this is dangerous

A model trained on this data can achieve 96% accuracy by always predicting "No Failure" — without learning anything meaningful. This is called the accuracy paradox and makes the model completely useless for detecting real failures.

🎤 Viva Line Class imbalance biases the model toward the majority class, reducing its ability to detect failures — the most important outcome.
05 — Solution

SMOTE — Fixing the Imbalance

Why KNN has high recall?

KNN finds the 5 nearest neighbors in feature space. After normalization, it catches almost all failures (98.7% recall) but has lower precision — more false alarms.

Why Random Forest wins?

RF builds 100 decision trees on random subsets and aggregates their votes. Ensemble learning reduces variance and avoids overfitting compared to a single tree.

🎤 Viva Line We evaluated models using accuracy, precision, recall, and F1-score — because accuracy alone is misleading on failure detection tasks.
